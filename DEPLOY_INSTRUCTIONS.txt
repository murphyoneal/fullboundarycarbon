      /docs/DRL-Configuration-as-Authority.pdf
      /docs/DRL-Configuration-as-Authority.docx
        Was: 17 pages, 167 KB — v1.0, never put through the
             same legal-redline pass the Reagan paper received.

             (4x), denominations (3x), tithe loop (3x), religion

        Now: 15 pages, 143 KB — v1.1 (May 2026, legal-review pass
             applied). Religious-authority frame recast as the

                  tithe, congregation, religious, religion, or
                  sacred. "State action doctrine" appears once in

    Defects fixed:
      1. methodology.html had class="active" on the About nav
         link. Removed.
      2. pages/behind-the-curtain.html had class="active" on the
         Corrected Ledger nav link. Removed.
      3. companions.html download cards each had a duplicated
         <h2> for the file title. Replaced with
         <p class="download-title">. Same fix applied to single
         download cards on codex.html and feedback.html.
      4. codex.html previously said "Three companion documents
         extend the Codex" and listed three. There are five.
         Now lists all five.
      5. Page counts and file sizes on companions.html,
         codex.html, and index.html have been updated to match
         the corrected document files in Category 1 above.

    Orphan-page fix:
      Methodology, Feedback, and Legal pages had no inbound links
      from anywhere on the site. A secondary footer nav has been
      added to every page (under the existing footer prose):

        Methodology . Feedback . Legal

      Smaller monospaced row, separated by a dashed rule from the
      main footer text. The masthead nav is unchanged.

    SEO and discoverability infrastructure:
      Every page now carries:
        - canonical URL pointing to fullboundarycarbon.org (not
          the .pages.dev fallback) — prevents Google indexing
          the same page under two URLs and splitting authority
        - Open Graph tags (og:title, og:description, og:url,
          og:type, og:site_name) — link previews on LinkedIn,
          X, Bluesky, Slack, and email will now show proper
          title/description cards instead of plain URLs
        - Twitter Card tags
        - favicon link

      Codex page additionally carries a JSON-LD ScholarlyArticle
      block so Google Scholar and structured-data crawlers can
      recognise the Codex as a citable academic work. Home page
      carries a JSON-LD WebSite block.

      New root files:
        - sitemap.xml — every public page, with priorities
          and lastmod dates
        - robots.txt — allow all crawlers, point to sitemap
        - 404.html — branded not-found page

    Favicon:
      Minimal SVG monogram ("DRL" in oxblood on cream) served
      from /assets/favicon.svg. Browsers will show it in tabs
      and bookmarks. SVG so it scales to any pixel density.

HOW TO DEPLOY (GitHub Desktop, one-shot)
----------------------------------------
Same procedure as v2.1 / v2.2. Repo at:
    C:\Users\<you>\Documents\GitHub\fullboundarycarbon\

  Step 1. Open File Explorer to your local repo folder.

  Step 2. Delete (or move out) everything currently in that folder
          EXCEPT the .git folder.

  Step 3. Extract site-v2-3.zip on your computer. Open the extracted
          site-v2-3 folder so you can see the contents.

  Step 4. Select ALL the contents of the extracted site-v2-3 folder
          (Ctrl+A), copy (Ctrl+C), paste (Ctrl+V) into your GitHub
          repo local folder.

  Step 5. Open GitHub Desktop. It will show many changes — additions
          (sitemap.xml, robots.txt, 404.html, assets/favicon.svg if
          not already deployed) and modifications to multiple .html
          files plus three replaced files in /docs/.

  Step 6. In the Summary box at the bottom type:
              "Site v2.3: legal redline applied; HTML hygiene; SEO"
          Click "Commit to main".

  Step 7. Click "Push origin" at the top.

  Step 8. Cloudflare auto-deploys within ~60 seconds. Test at:
              https://fullboundarycarbon.pages.dev
